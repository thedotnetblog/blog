أدوات Azure MCP مدمجة الآن في Visual Studio 2022 — لا حاجة إلى امتداد

تُشحن أدوات Azure MCP كجزء من حمل عمل تطوير Azure في Visual Studio 2022. أكثر من 230 أداة، و45 خدمة Azure، وصفر امتدادات للتثبيت.

تمت ترجمة هذا المقال تلقائياً. للنسخة الأصلية، انقر هنا.

إذا كنت تستخدم أدوات Azure MCP في Visual Studio من خلال الامتداد المنفصل، فأنت تعرف العملية — تثبيت VSIX، إعادة التشغيل، والأمل ألا ينكسر، وإدارة تعارضات الإصدارات. لقد اختفى هذا الاحتكاك.

أعلنت Yun Jung Choi أن أدوات Azure MCP تُشحن الآن مباشرةً كجزء من حمل عمل تطوير Azure في Visual Studio 2022. بدون امتداد. بدون VSIX. بدون رقصة إعادة التشغيل.

ما الذي يعنيه هذا فعلاً

ابتداءً من Visual Studio 2022 الإصدار 17.14.30، يأتي Azure MCP Server مجمّعاً مع حمل عمل تطوير Azure. إذا كان حمل العمل هذا مثبتاً بالفعل، فتحتاج فقط إلى تفعيله في GitHub Copilot Chat وستنتهي.

أكثر من 230 أداة عبر 45 خدمة Azure — متاحة مباشرةً من نافذة الدردشة. سرّد حسابات التخزين لديك، انشر تطبيق ASP.NET Core، شخّص مشاكل App Service، استعلم Log Analytics — كل ذلك دون فتح أي علامة تبويب في المتصفح.

لماذا هذا أهم مما يبدو

إليك الأمر في أدوات المطورين: كل خطوة إضافية هي احتكاك، والاحتكاك يقتل الاستخدام. وجود MCP كامتداد منفصل كان يعني تعارض الإصدارات وفشل التثبيت وشيء آخر لتبقيه محدثاً. إدماجه في نظام حمل العمل يعني:

  • مسار تحديث واحد عبر Visual Studio Installer
  • لا انجراف في الإصدارات بين الامتداد وبيئة التطوير المتكاملة
  • دائماً حديث — يتحدث MCP Server مع إصدارات VS المنتظمة

بالنسبة للفرق التي تعتمد معياراً على Azure، هذا أمر مهم. تثبّت حمل العمل مرة واحدة، تفعّل الأدوات، وتكون موجودة في كل جلسة.

ما الذي يمكنك فعله به

تغطي الأدوات دورة التطوير الكاملة من خلال Copilot Chat:

  • التعلم — اسأل عن خدمات Azure وأفضل الممارسات وأنماط البنية
  • التصميم والتطوير — احصل على توصيات الخدمة وضبط كود التطبيق
  • النشر — زوّد الموارد وانشر مباشرةً من بيئة التطوير المتكاملة
  • استكشاف الأخطاء — استعلم السجلات وتحقق من صحة الموارد وشخّص مشاكل الإنتاج

مثال سريع — اكتب هذا في Copilot Chat:

List my storage accounts in my current subscription.

يستدعي Copilot أدوات Azure MCP في الخلفية، ويستعلم اشتراكاتك، ويعيد قائمة منسقة بالأسماء والمواقع وحيازات SKU. لا حاجة إلى البوابة الإلكترونية.

كيفية التفعيل

  1. حدّث إلى Visual Studio 2022 17.14.30 أو أحدث
  2. تأكد من تثبيت حمل عمل Azure development
  3. افتح GitHub Copilot Chat
  4. انقر على زر Select tools (أيقونة المفكَّين)
  5. فعّل Azure MCP Server

هذا كل شيء. يبقى مفعّلاً عبر الجلسات.

تحفظ واحد

الأدوات معطلة افتراضياً — تحتاج إلى الاشتراك الصريح. والأدوات الخاصة بـ VS 2026 غير متاحة في VS 2022. كما يعتمد توافر الأدوات على أذونات اشتراك Azure لديك، تماماً كما في البوابة الإلكترونية.

الصورة الأكبر

هذا جزء من اتجاه واضح: يصبح MCP الطريقة المعيارية لعرض أدوات السحابة في بيئات تطوير المطورين. رأينا بالفعل إصدار Azure MCP Server 2.0 المستقر وتكاملات MCP عبر VS Code والمحررات الأخرى. دمجه في نظام حمل العمل في Visual Studio هو التطور الطبيعي.

بالنسبة لنا كمطوري .NET الذين يعيشون داخل Visual Studio، يزيل هذا سبباً آخر للتحول إلى بوابة Azure. وبصراحة، كلما قل التبديل بين علامات التبويب، كان ذلك أفضل.
