Écris pour The .NET Blog

Partage tes connaissances avec la communauté .NET.

The .NET Blog est une publication communautaire où les développeurs partagent des analyses, des tutoriels et des histoires sur .NET, Azure, l’IA et le développement cloud-native. Nous accueillons les contributions de développeurs de tous niveaux — que vous écriviez votre premier article technique ou que vous soyez un conférencier expérimenté.

Comment rejoindre

Tout se passe sur GitHub avec un workflow de pull requests. Voici comment commencer :

1. Forker le dépôt

Rendez-vous sur github.com/thedotnetblog/blog et forkez le dépôt sur votre compte GitHub.

2. Créer votre profil d’auteur

Si c’est votre première contribution, ajoutez-vous comme auteur en créant un dossier dans content/authors/your-username/ avec un fichier index.md contenant votre nom, rôle et biographie. Ajoutez votre image d’avatar (carrée, au moins 200x200px) dans static/img/authors/.

3. Rédiger votre article

Créez un nouveau dossier dans content/posts/your-username/your-post-slug/ et ajoutez un fichier index.md avec le titre, la date, l’auteur, la description et les tags.

4. Ouvrir une Pull Request

Poussez vos modifications sur votre fork et ouvrez une pull request vers la branche main. Notre équipe la passera en revue et fournira des commentaires dans les quelques jours.

Ce que nous recherchons

  • Tutoriels — guides pas à pas sur .NET, Azure, IA, Blazor, Aspire et plus
  • Analyses approfondies — explorations détaillées d’une technologie, d’un pattern ou d’une architecture
  • Histoires de la communauté — votre expérience avec .NET en production
  • Résumés d’événements — comptes-rendus de conférences, meetups ou webinaires

Directives

  • Le contenu doit être technique et pertinent pour l’écosystème .NET
  • Les exemples de code doivent être exacts et testés dans un vrai projet
  • Inclure une description significative et au moins un tag pertinent
  • Les articles sont automatiquement traduits dans toutes les langues supportées

Contact

Ouvrez une issue sur GitHub ou contactez-nous sur X / Twitter. Nous serions ravis de vous accueillir dans la communauté !
