The .NET Blog 是一个由社区驱动的出版物，开发者们在这里分享关于 .NET、Azure、AI 和云原生开发的见解、教程和故事。**我们欢迎各个水平的开发者投稿** — 无论你是在写第一篇技术文章，还是经验丰富的演讲者。

## 如何加入

所有内容都在 GitHub 上，采用 Pull Request 工作流。以下是入门步骤：

### 1. Fork 仓库

访问 [github.com/thedotnetblog/blog](https://github.com/thedotnetblog/blog) 并将其 Fork 到你的 GitHub 账户。

### 2. 创建作者简介

如果这是你的第一次投稿，请在 `content/authors/your-username/` 下创建文件夹并添加 `index.md` 文件，包含你的姓名、职位、简介和社交链接。将头像图片（正方形，至少 200×200px）添加到 `static/img/authors/`。

### 3. 撰写文章

在 `content/posts/your-username/your-post-slug/` 下创建文件夹并添加 `index.md`，包含标题、日期、作者、描述和标签。

### 4. 提交 Pull Request

将更改推送到 Fork，并向 `main` 分支发起 Pull Request。我们的团队会在几天内审阅并提供反馈。

## 我们在寻找什么

- **教程** — .NET、Azure、AI、Blazor、Aspire 等的分步指南
- **深度解析** — 对技术、模式或架构的深入探讨
- **社区故事** — 你在生产环境中使用 .NET 的经验
- **活动回顾** — 会议、Meetup 或网络研讨会的总结

## 规范

- 内容应与 .NET 生态系统相关且具有技术性
- 代码示例必须准确且在真实项目中经过测试
- 包含有意义的描述和至少一个相关标签
- 文章会自动翻译成所有支持的语言
