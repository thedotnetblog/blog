أطلقت Microsoft للتو [SQLCon 2026 جنباً إلى جنب مع FabCon في أتلانتا](https://www.microsoft.com/en-us/sql-server/blog/2026/03/18/advancing-agentic-ai-with-microsoft-databases-across-a-unified-data-estate/). سأتجاوز شرائح أسعار المؤسسات وأركز على الأجزاء المهمة للمطورين الذين يبنون مع Azure SQL والذكاء الاصطناعي.

## خادم SQL MCP في المعاينة العامة

هذا هو العنوان الرئيسي بالنسبة لي. يحتوي Azure SQL Database Hyperscale الآن على **خادم SQL MCP** في المعاينة العامة يتيح لك ربط بيانات SQL بأمان بوكلاء الذكاء الاصطناعي وCopilots باستخدام [بروتوكول سياق النموذج](https://modelcontextprotocol.io/).

لمطوري .NET الذين يبنون وكلاء ذكاء اصطناعي باستخدام Semantic Kernel أو Microsoft Agent Framework، يفتح هذا مساراً للتكامل. وكيلك يحتاج إلى فحص المخزون؟ البحث عن سجل عميل؟ التحقق من صحة طلب؟ يمنحه MCP طريقة منظمة للقيام بذلك دون كتابة كود جلب البيانات المخصص.

## GitHub Copilot في SSMS 22 متاح الآن بشكل عام

إذا كنت تقضي وقتاً في SQL Server Management Studio، فإن GitHub Copilot متاح الآن بشكل عام في SSMS 22. نفس تجربة Copilot التي تستخدمها في VS Code وVisual Studio، لكن لـ T-SQL.

القيمة العملية واضحة: مساعدة قائمة على المحادثة لكتابة الاستعلامات، وإعادة هيكلة الإجراءات المخزنة، واستكشاف مشاكل الأداء.

## تحسينات جوهرية على الفهارس المتجهة

يحتوي Azure SQL Database الآن على فهارس متجهة أسرع وأكثر قدرة مع دعم كامل للإدراج والتحديث والحذف. هذا يعني أن بيانات المتجه تبقى محدّثة في الوقت الفعلي — لا حاجة لإعادة الفهرسة الدفعية.

ما الجديد:
- **الكميّة** لأحجام فهرس أصغر دون فقدان كبير في الدقة
- **التصفية التكرارية** لنتائج أكثر دقة
- **تكامل أوثق مع محسّن الاستعلام** لأداء متوقع

إذا كنت تجري RAG مع Azure SQL كمخزن متجهات، فهذه التحسينات مفيدة بشكل مباشر.

## مركز قواعد البيانات في Fabric: الإدارة الوكيلية

أعلنت Microsoft عن **مركز قواعد البيانات في Microsoft Fabric** (الوصول المبكر) الذي يوفر لوحة زجاجية واحدة عبر Azure SQL وCosmos DB وPostgreSQL وMySQL وSQL Server عبر Arc.

تراقب وكلاء الذكاء الاصطناعي بيئة قواعد البيانات باستمرار، وتكشف ما تغيّر، وتشرح لماذا يهم، وتقترح ما يجب فعله لاحقاً.

## ماذا يعني هذا لمطوري .NET

1. **جرّب خادم SQL MCP** إذا كنت تبني وكلاء ذكاء اصطناعي.
2. **فعّل Copilot في SSMS** إذا لم تفعل ذلك بعد.
3. **ابحث في الفهارس المتجهة** إذا كنت تجري RAG وتشغّل مخزن متجهات منفصلاً.
