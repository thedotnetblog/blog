Знаете это сообщение «Доступна новая версия azd», которое появляется каждые несколько недель? То самое, которое вы закрываете, потому что не помните, устанавливали ли вы `azd` через winget, Homebrew или тот curl-скрипт, который запустили полгода назад? Так вот, это наконец-то исправлено.

Microsoft выпустила [`azd update`](https://devblogs.microsoft.com/azure-sdk/azd-update/) — единую команду, которая обновляет Azure Developer CLI до последней версии независимо от того, как вы её изначально установили. Windows, macOS, Linux — неважно. Одна команда.

## Как это работает

```bash
azd update
```

Вот и всё. Если хотите ранний доступ к новым возможностям, можно переключиться на ежедневную Insiders-сборку:

```bash
azd update --channel daily
azd update --channel stable
```

Команда определяет текущий метод установки и использует подходящий механизм обновления под капотом. Больше никаких «подожди, я на этой машине winget или choco использовал?»

## Небольшой нюанс

`azd update` доступна начиная с версии 1.23.x. Если у вас более старая версия, придётся выполнить последнее ручное обновление через исходный метод установки. После этого `azd update` берёт всё на себя.

Проверьте текущую версию командой `azd version`. Если нужна чистая установка, [документация по установке](https://learn.microsoft.com/azure/developer/azure-developer-cli/install-azd) вам поможет.

## Почему это важно

Это небольшое улучшение качества жизни, но для тех из нас, кто ежедневно использует `azd` для деплоя ИИ-агентов и Aspire-приложений в Azure, быть на последней версии означает меньше моментов «этот баг уже был исправлен в последнем релизе». На одну заботу меньше.
