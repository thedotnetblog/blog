إذا كنت تتابع رحلة Agent Framework منذ الأيام الأولى لـ Semantic Kernel وAutoGen، فهذا الخبر مهم. وصل Microsoft Agent Framework للتو إلى [الإصدار 1.0](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/) — جاهز للإنتاج، واجهات برمجية مستقرة، والتزام بالدعم طويل الأمد. متاح لـ .NET وPython، وجاهز فعلاً لأعباء العمل الحقيقية.

دعنا نتجاوز ضجيج الإعلان ونركز على ما يهم إذا كنت تبني تطبيقات مدعومة بالذكاء الاصطناعي مع .NET.

## النسخة المختصرة

يوحّد Agent Framework 1.0 ما كان يُعرف بـ Semantic Kernel وAutoGen في SDK واحد مفتوح المصدر. تجريد وكيل واحد. محرك تنسيق واحد. مزودو ذكاء اصطناعي متعددون. إذا كنت تتأرجح بين Semantic Kernel لأنماط المؤسسات وAutoGen لسير عمل الوكلاء المتعددين على مستوى البحث، يمكنك التوقف الآن. هذا هو SDK الوحيد الآن.

## البدء بسيط بشكل مثير للاهتمام

إليك وكيلاً عاملاً في .NET:

```csharp
// dotnet add package Microsoft.Agents.AI.OpenAI --prerelease
using Microsoft.Agents.AI;
using Microsoft.Agents.AI.Foundry;
using Azure.Identity;

var agent = new AIProjectClient(endpoint: "https://your-project.services.ai.azure.com")
    .GetResponsesClient("gpt-5.3")
    .AsAIAgent(
        name: "HaikuBot",
        instructions: "You are an upbeat assistant that writes beautifully."
    );

Console.WriteLine(await agent.RunAsync("Write a haiku about shipping 1.0."));
```

هذا كل شيء. بضعة أسطر ولديك وكيل ذكاء اصطناعي يعمل على Azure Foundry. المكافئ في Python موجز بالقدر نفسه. أضف أدوات الوظائف والمحادثات متعددة الأدوار والبث أثناء تقدمك — سطح API يتوسع دون أن يصبح غريباً.

## تنسيق الوكلاء المتعددين — هذا هو الأمر الحقيقي

الوكلاء الفرديون مناسبون للعروض التوضيحية، لكن سيناريوهات الإنتاج عادةً ما تحتاج إلى تنسيق. يُشحن Agent Framework 1.0 مع أنماط تنسيق مُختبرة في ميدان القتال مباشرة من Microsoft Research وAutoGen:

- **Sequential** — يعالج الوكلاء بالتسلسل (كاتب → مراجع → محرر)
- **Concurrent** — التوزيع على وكلاء متعددين بالتوازي، وتقارب النتائج
- **Handoff** — يفوّض وكيل إلى آخر بناءً على النية
- **Group chat** — يناقش وكلاء متعددون ويتقاربون نحو حل
- **Magentic-One** — نمط الوكلاء المتعددين على مستوى البحث من MSR

جميعها تدعم البث، ونقاط التفتيش، وموافقات human-in-the-loop، والإيقاف المؤقت/الاستئناف. جزء نقاط التفتيش مهم — تنجو سير العمل طويلة الأمد من إعادة تشغيل العمليات.

## الميزات الأكثر أهمية

**خطافات Middleware.** تعرف خطوط أنابيب middleware في ASP.NET Core؟ نفس المفهوم، لكن لتنفيذ الوكلاء. اعترض كل مرحلة — أضف أمان المحتوى والتسجيل وسياسات الامتثال — دون لمس مطالبات الوكلاء.

**الذاكرة القابلة للتوصيل.** سجل المحادثات، والحالة الدائمة من المفتاح إلى القيمة، والاسترداد القائم على المتجهات. اختر نظامك الخلفي: Foundry Agent Service أو Mem0 أو Redis أو Neo4j أو اصنع نظامك الخاص.

**وكلاء YAML التعريفيون.** عرّف تعليمات وكيلك وأدواته وذاكرته وتوبولوجيا التنسيق في ملفات YAML متحكم فيها بالإصدار. حمّل وشغّل باستدعاء API واحد.

**دعم A2A وMCP.** يتيح MCP (بروتوكول سياق النموذج) للوكلاء اكتشاف الأدوات الخارجية واستدعاءها ديناميكياً. يُمكّن A2A (بروتوكول من وكيل إلى وكيل) التعاون عبر بيئات التشغيل.

## ميزات المعاينة الجديرة بالمتابعة

- **DevUI** — مُصحح أخطاء محلي يعتمد على المتصفح لتصور تنفيذ الوكيل وتدفقات الرسائل واستدعاءات الأدوات في الوقت الفعلي.
- **GitHub Copilot SDK وClaude Code SDK** — استخدم Copilot أو Claude كسخام وكيل مباشرةً من كود التنسيق الخاص بك.
- **Agent Harness** — بيئة تشغيل محلية قابلة للتخصيص تمنح الوكلاء الوصول إلى الصدفة ونظام الملفات وحلقات المراسلة.
- **Skills** — حزم إمكانات نطاق قابلة لإعادة الاستخدام.

## الهجرة من Semantic Kernel أو AutoGen

إذا كان لديك كود Semantic Kernel أو AutoGen موجود، فهناك مساعدو هجرة مخصصون يحللون كودك ويولّدون خطط هجرة خطوة بخطوة.

## خلاصة

Agent Framework 1.0 هو معلم الإنتاج الذي كانت فرق المؤسسات تنتظره. واجهات برمجية مستقرة، ودعم متعدد المزودين، وأنماط تنسيق تعمل فعلاً على النطاق الكبير.

الإطار [مفتوح المصدر بالكامل على GitHub](https://github.com/microsoft/agent-framework). إذا كنت تنتظر إشارة "آمن للاستخدام في الإنتاج" — ها هي.
