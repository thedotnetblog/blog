Visual Studio только что получила самое значительное обновление Copilot. Mark Downie [объявил о мартовском релизе](https://devblogs.microsoft.com/visualstudio/visual-studio-march-update-build-your-own-custom-agents/), и заголовок — пользовательские агенты, но честно говоря, инструмент `find_symbol` может оказаться функцией, которая больше всего изменит ваш рабочий процесс.

## Пользовательские агенты Copilot в вашем репозитории

Хотите, чтобы Copilot следовал стандартам кодирования вашей команды? Пользовательские агенты определяются как файлы `.agent.md` в `.github/agents/`. Каждый агент имеет полный доступ к workspace, пониманию кода, инструментам, вашей предпочтительной модели и MCP-соединениям.

## Навыки агента: переиспользуемые наборы инструкций

Навыки автоматически подхватываются из `.github/skills/` в вашем репозитории или `~/.copilot/skills/` в профиле.

## find_symbol: навигация с поддержкой языка

Новый инструмент `find_symbol` даёт агентному режиму Copilot навигацию по символам на основе языковых сервисов. Вместо текстового поиска агент может находить все ссылки на символ и получать доступ к информации о типах и области видимости.

Для .NET-разработчиков это огромное улучшение — кодовые базы на C# с глубокими иерархиями типов получают огромную пользу.

## Профилирование тестов с Copilot

В контекстном меню Test Explorer появился **Profile with Copilot**. Profiling Agent запускает тест и автоматически анализирует производительность.

## Perf Tips во время отладки

Оптимизация производительности теперь происходит прямо во время отладки. Visual Studio показывает время выполнения inline. Увидели медленную строку? Кликните Perf Tip и попросите Copilot предложить оптимизации.

## Исправление уязвимостей NuGet из Solution Explorer

При обнаружении уязвимости NuGet-пакета в Solution Explorer появляется ссылка **Fix with GitHub Copilot**.

Пользовательские агенты и навыки — заголовок, но `find_symbol` — скрытая жемчужина, которая фундаментально меняет точность Copilot при рефакторинге .NET-кода. Скачайте [Visual Studio 2026 Insiders](https://visualstudio.microsoft.com/downloads/), чтобы всё попробовать.
