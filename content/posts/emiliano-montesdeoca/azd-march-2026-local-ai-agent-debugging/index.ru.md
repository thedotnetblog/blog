Семь релизов за месяц. Именно столько выпустила команда Azure Developer CLI (`azd`) в марте 2026, и главная фича — та, которую я ждал: **локальный цикл запуска и отладки для ИИ-агентов**.

## Запуск и отладка ИИ-агентов без деплоя

Это главное. Новое расширение `azure.ai.agents` добавляет команды для полноценного inner-loop для ИИ-агентов:

- `azd ai agent run` — запускает агент локально
- `azd ai agent invoke` — отправляет сообщения (локально или в продакшн)
- `azd ai agent show` — показывает статус контейнера и здоровье
- `azd ai agent monitor` — стримит логи контейнера в реальном времени

Раньше тестирование ИИ-агента означало деплой в Microsoft Foundry при каждом изменении. Теперь можно итерировать локально, тестировать поведение агента и деплоить только когда готов.

## GitHub Copilot настраивает ваш azd-проект

`azd init` теперь предлагает опцию "Set up with GitHub Copilot (Preview)". Вместо ручного ответа на промпты, Copilot-агент генерирует конфигурацию за вас. Когда команда падает, `azd` предлагает ИИ-ассистированное устранение неполадок — всё не выходя из терминала.

## Container App Jobs и улучшения деплоя

- **Container App Jobs**: `azd` теперь деплоит `Microsoft.App/jobs` через существующую конфигурацию `host: containerapp`.
- **Настраиваемые таймауты деплоя**: Новый флаг `--timeout` в `azd deploy` и поле `deployTimeout` в `azure.yaml`.
- **Фолбэк удалённой сборки**: При неудаче ACR-сборки `azd` автоматически переключается на локальный Docker/Podman.
- **Локальная preflight-валидация**: Bicep-параметры проверяются локально перед деплоем.

## Улучшения DX

- **Автоматическое определение pnpm/yarn** для JS/TS-проектов
- **Поддержка pyproject.toml** для Python
- **Локальные каталоги шаблонов** — `azd init --template` принимает пути файловой системы
- **Улучшенные сообщения об ошибках** в режиме `--no-prompt`
- **Переменные окружения сборки** — инжектируются во все подпроцессы сборки (.NET, Node.js, Java, Python)
