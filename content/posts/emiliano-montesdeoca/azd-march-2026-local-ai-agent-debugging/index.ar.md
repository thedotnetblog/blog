سبعة إصدارات في شهر واحد. هذا ما دفعه فريق Azure Developer CLI (`azd`) في مارس 2026، والميزة الرئيسية هي ما كنت أنتظره: **حلقة تشغيل وتصحيح أخطاء محلية لوكلاء الذكاء الاصطناعي**.

## تشغيل وكلاء الذكاء الاصطناعي وتصحيح أخطائها دون نشر

هذا هو الكبير. تُضيف امتداد `azure.ai.agents` الجديد مجموعة من الأوامر:

- `azd ai agent run` — يشغّل الوكيل محلياً
- `azd ai agent invoke` — يرسل رسائل إليه (محلياً أو منشوراً)
- `azd ai agent show` — يعرض حالة الحاوية وصحتها
- `azd ai agent monitor` — يبث سجلات الحاوية في الوقت الفعلي

في السابق، كان اختبار وكيل الذكاء الاصطناعي يعني النشر إلى Microsoft Foundry في كل مرة. الآن يمكنك التكرار محلياً.

## GitHub Copilot يُسقّل مشروع azd الخاص بك

يقدم `azd init` الآن خيار "إعداد مع GitHub Copilot (معاينة)". يُسقّل وكيل Copilot التهيئة لبنية مشروعك.

## Container App Jobs وتحسينات النشر

- **Container App Jobs**: ينشر `azd` الآن `Microsoft.App/jobs` عبر تهيئة `host: containerapp` الموجودة
- **مهل نشر قابلة للتهيئة**: علامة `--timeout` الجديدة على `azd deploy`
- **احتياطي بناء محلي**: عند فشل البناء عن بُعد، يعود `azd` تلقائياً إلى البناء المحلي
- **التحقق المسبق المحلي**: تتحقق معاملات Bicep محلياً قبل النشر
