Если вы используете Azure Boards, вы наверняка сталкивались с этим: читаете описание рабочего элемента, может проверяете критерии приёмки, и случайно делаете двойной клик. Бум — вы в режиме редактирования. Вы ничего не хотели редактировать. Просто читали.

Dan Hellem [объявил об исправлении](https://devblogs.microsoft.com/devops/improving-the-markdown-editor-for-work-items/), и это одно из тех изменений, которые звучат мелко, но реально убирают трение из повседневного рабочего процесса.

## Что изменилось

Markdown-редактор для текстовых полей рабочих элементов теперь открывается в **режиме просмотра по умолчанию**. Можно читать контент и взаимодействовать с ним — переходить по ссылкам, проверять форматирование — не беспокоясь о случайном входе в режим редактирования.

Когда действительно хотите редактировать, нажмите на иконку редактирования вверху поля. Закончили — явно вернитесь в режим просмотра. Просто, намеренно, предсказуемо.

## Почему это важнее, чем кажется

[Тред обратной связи от сообщества](https://developercommunity.visualstudio.com/t/Markdown-editor-for-work-item-multi-line/10935496) был длинным. Поведение двойного клика для редактирования было введено с Markdown-редактором в июле 2025, и жалобы начались почти сразу.

Для команд, которые делают планирование спринтов, уточнение бэклога или код-ревью в Azure Boards, такое микро-трение накапливается.

## Статус развёртывания

Уже раскатывается для части пользователей и расширится на всех в ближайшие две-три недели.

## Подведём итоги

Не каждое улучшение должно быть заголовочной фичей. Иногда лучшее обновление — просто убрать что-то раздражающее. Это именно такой случай — маленький UX-фикс, который делает Azure Boards менее враждебным для людей, которые просто хотят спокойно прочитать свои рабочие элементы.
